Documentação dos Endpoints

Método Caminho da URL Descrição Parâmetros
POST /ctoclient Cadastra o cliente no banco. Nenhum.
GET /ctoclient Lista todos os clientes no banco. Nenhum.
GET /ctoclientlog Lista todos os logs clientes no banco. Nenhum.
POST /ctoclientid Recebe o cto_id e o nome e retorna o true ou false.
  • cto_id: número do cto_id do cliente.
  • name: nome do cliente.
POST /updatectoclient Recebe dados e atualiza no banco. Nenhum.
DELETE /deletectolog/:id Deleta um log de cliente.
  • id: ID do log a ser deletado.
DELETE /deletectoclient/:id Deleta um cliente do banco.
  • id: ID do cliente a ser deletado.
GET /users Lista todos os usuários do banco. Nenhum.
POST /users Cadastra um usuário no banco.
  • nome: nome do usuário.
  • email: email do usuário.
  • senha: senha do usuário.
PUT /users/:id Atualiza as informações de um usuário pelo ID.
  • id: ID do usuário a ser atualizado.
  • nome: novo nome do usuário.
  • email: novo email do usuário.
  • senha: nova senha do usuário.
DELETE /users/:id Deleta um usuário do banco pelo ID.
  • id: ID do usuário a ser deletado.
GET /users/:id Lista um usuário pelo ID.
  • id: ID do usuário a ser listado.
POST /login Realiza o login do usuário.
  • email: email do usuário.
  • senha: senha do usuário.
POST /vlan Salva a VLAN no banco. Nenhum.
GET /vlan Lista as VLANs no banco. Nenhum.
POST /tracking Cadastra o tracking no banco. Nenhum.
POST /trackinget Lista o tracking no banco. Nenhum.
GET /tomodat Fetch direto do Tomodat (a cada 60 segundos). Nenhum.
POST /client Adiciona um cliente no banco, registro de log e Tomodat. Nenhum.
GET /logctoclient Lista os logs de clientes cadastrados nas CTOS. Nenhum.
DELETE /deleteclientfromtomodat/:id Deleta um cliente no servidor do Tomodat.
  • id: ID do cliente a ser deletado.
POST /instalacoes Salva no banco os dados da instalação. Nenhum.
GET /instalacoesget Requisita do banco os dados da instalação. Nenhum.
GET /fetch Lista o fetch do banco. Nenhum.
GET /fetchwithctoclient Pega o fetch no banco e agrega com as localizações já cadastradas dos clientes. Nenhum.
POST /equipamentcadastro Salva no banco os dados do equipamento. Nenhum.
GET /equipamentget Lista os dados dos equipamentos. Nenhum.
PUT /equipamentatualizar Atualiza os dados do equipamento.
  • id: ID do equipamento a ser atualizado.
  • name: novo nome do equipamento.
  • ip: novo IP do equipamento.
  • category: nova categoria do equipamento.
DELETE /equipamentdelete Deleta um equipamento.
  • id: ID do equipamento a ser deletado.
POST /pppoecadastro Salva no banco os dados do PPPoE. Nenhum.
GET /pppoeget Requisita do banco os dados do PPPoE. Nenhum.
POST /pppoeonline Verifica se o PPPoE está online. Nenhum.
GET /getallpppoeonline Verifica quais PPPoEs estão online no concentrador e retorna para o frontend. Nenhum.
